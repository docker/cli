# Security Policy

## Supported Versions
| Version | Supported |
|---------|-----------|
| 2.x     | ✅ |
| 1.x     | ❌ |

We only provide security updates for the latest major version.

## Reporting a Vulnerability
If you discover a security issue, please report it responsibly:

- Use [GitHub Security Advisories](https://github.com/OWNER/REPO/security/advisories)  
- Or email us at: matysekotlant@gmail.com  

We will respond within **48 hours**. Please include as much detail as possible (steps to reproduce, affected versions, potential impact).

## Process
- Vulnerabilities are triaged immediately.
- Fixes are developed in a private branch.
- Security patches are released before public disclosure.
- Dependencies are monitored with **Dependabot**.
- Code is scanned regularly with **CodeQL**.
- Contributors are required to use **Multi-Factor Authentication (MFA)**.

## Disclosure Policy
We follow a responsible disclosure process:
- Confirmed vulnerabilities will be announced via GitHub Security Advisories.
- Fixes will be documented in the changelog.
- Public disclosure happens only after a patch is available.
